In fact, we can be even more efficient. The condition in a conditional section doesn't have to be true or false – instead it is evaluated as truthy or falsy.

Let's stipulate that the user property only exists if a user is signed in, and that if they are signed in, user is an object with a name property.

Edit the sign-in code accordingly:

ractive.on( 'signIn', function () {
  var name = prompt( 'Enter your username to sign in', 'ractive_fan' );
  ractive.set( 'user.name', name );
});

Here, we're setting a property on an object that doesn't exist yet. That's fine – Ractive.js will create the object. If we'd done ractive.set( 'user[0]', name ) or ractive.set( 'user.0', name ) it would assume we wanted to create an array instead of an object.

Initially, user will be undefined, but once someone signs in, user will be an object. Since undefined is falsy and objects are always truthy, we can use user instead of signedIn.

Update the template accordingly, discarding signedIn in favour of user. Notice that {{#user}} isn't just a conditional section any more – it's also a context section. So we can use the {{name}} property in our 'welcome back!' message.

Execute the code.

In JavaScript, the following values are falsy: undefined, null, NaN, 0, "" (the empty string), and false.

However in mustache, there is a special case – arrays with a length of 0 are treated as falsy. This will make sense when we tackle list sections in the next tutorial.
